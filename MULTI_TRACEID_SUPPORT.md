# 交易类型追踪 - 多 TraceID 分组支持

**版本**: v2.3.0  
**发布日期**: 2026-04-09

---

## 📋 功能概述

**交易类型追踪** 页面现在支持同一个 REQ_SN 对应多个不同 TraceID 的场景（重复查询）。

### 场景说明

商户可能发起重复查询，同一个 REQ_SN 对应多个不同的 TraceID：

```
REQ_SN: 202604080800000001
  ├─ TraceID: TC5PCfGK (第一次查询)
  │   └─ 应用链路：aipg → trxqry → pay (15 条日志)
  └─ TraceID: TC07bG2Y (重复查询)
      └─ 应用链路：aipg → trxqry → pay (12 条日志)
```

---

## 🎯 核心功能

### 1. 自动检测重复查询

系统会自动检测同一个 REQ_SN 是否对应多个 TraceID。

### 2. 分组展示

使用**折叠面板**按 TraceID 分组展示：

```
▼ TraceID: TC5PCfGK  27 条日志 | 1 次 REQ_SN 匹配  2026-04-08 09:00:00.304
  ┌─────────────────────────────────────────────────┐
  │ [1] aipg → [2] trxqry → [3] pay                │
  │  15 条       8 条        4 条                    │
  ├─────────────────────────────────────────────────┤
  │ Tab: aipg | trxqry | pay                        │
  │ [表格显示该 TraceID 在各应用的日志]               │
  └─────────────────────────────────────────────────┘

▶ TraceID: TC07bG2Y  22 条日志 | 1 次 REQ_SN 匹配  2026-04-08 09:00:03.547
  [点击展开查看详情]
```

### 3. 按时间排序

多个 TraceID 分组按**第一次出现的时间**排序。

---

## 🖥️ 页面功能

### 查询条件

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 交易类型 | 下拉选择 | ✅ | 显示交易代码、名称和关联应用 |
| REQ_SN | 文本输入 | ✅ | 交易序列号 |
| 日志时间 | 文本输入 | ❌ | 格式：YYYYMMDDHH (10 位数字) |

### 结果展示

#### 多 TraceID 分组模式

- **折叠面板**: 每个 TraceID 一个面板
- **面板标题**: TraceID + 日志条数 + REQ_SN 匹配次数 + 时间
- **应用链路图**: 显示该 TraceID 在各应用的日志分布
- **应用 Tab**: 按应用分组查看日志详情
- **高亮显示**: REQ_SN 匹配的行特殊标记

#### 单 TraceID 模式

- 保持原有展示方式
- 直接显示应用链路图和日志详情

## 📊 查询逻辑

```
1. 用户选择交易类型 (如 310011)
   关联应用：sft-aipg → sft-trxqry → sft-pay
       ↓
2. 从入口应用 (sft-aipg) 查找所有包含 REQ_SN 的行
       ↓
3. 提取所有不同的 TraceID
   - TC5PCfGK (第一次查询)
   - TC07bG2Y (重复查询)
       ↓
4. 对每个 TraceID:
   - 依次查询所有关联应用的日志
   - 按应用分组
       ↓
5. 按 TraceID 第一次出现时间排序
       ↓
6. 分组展示（折叠面板）
```

## 📝 使用示例

### 示例 1: 单次查询

**输入**:
- 交易类型：`310011` (协议支付)
- REQ_SN: `20260408090000019466514799`
- 日志时间：`2026040809`

**结果**:
```
1 个 TraceID 分组
└─ TraceID: TCiDl7n3 (9 条日志)
   └─ 应用链路：sft-aipg → sft-trxqry → sft-pay
```

### 示例 2: 重复查询

**输入**:
- 交易类型：`310011` (协议支付)
- REQ_SN: `202604080800000001`
- 日志时间：`2026040809`

**结果**:
```
2 个 TraceID 分组
├─ TraceID: TC5PCfGK (27 条日志) - 2026-04-08 09:00:00.304
│   └─ 应用链路：sft-aipg (15) → sft-trxqry (8) → sft-pay (4)
└─ TraceID: TC07bG2Y (22 条日志) - 2026-04-08 09:00:03.547
    └─ 应用链路：sft-aipg (12) → sft-trxqry (7) → sft-pay (3)
```

---

## ⚠️ 注意事项

1. **应用顺序**: 关联应用的顺序很重要，第一个应用作为入口应用
2. **日志时间**: 建议指定日志时间，可以加快查询速度
3. **分组排序**: 多个 TraceID 按第一次出现的时间排序
4. **性能考虑**: 重复查询会增加查询量（每个 TraceID 都要查询所有应用）

---

## 🎨 UI 特性

- **折叠面板**: 清晰展示多个 TraceID 分组
- **分组信息**: TraceID + 日志数 + 匹配次数 + 时间
- **应用链路**: 每个分组独立显示应用链路图
- **Tab 切换**: 按应用查看日志详情
- **高亮显示**: REQ_SN 匹配的行特殊标记（浅蓝色背景）
- **展开详情**: 支持查看完整日志内容

---

## 🔄 与日志追踪查询的区别

| 功能 | 日志追踪查询 | 交易类型追踪 |
|------|-------------|-------------|
| **查询方式** | 选择单个应用 | 自动加载多个应用 |
| **应用数量** | 1 个 | 多个（配置决定） |
| **TraceID 提取** | 从选定应用提取 | 从入口应用提取 |
| **结果展示** | 按 TraceID 分组 | 按 TraceID 分组 + 按应用分组 |
| **适用场景** | 精确定位某应用 | 查看完整交易链路 |
