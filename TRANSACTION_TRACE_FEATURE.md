# 交易类型日志追踪功能

**版本**: v2.1.0  
**发布日期**: 2026-04-09

---

## 📋 功能概述

**交易类型日志追踪** 页面允许用户根据交易类型配置的关联应用，依次展示完整交易链路。

### 核心特性

- ✅ 选择交易类型自动加载应用列表
- ✅ 从入口应用（第一个应用）提取 TraceID
- ✅ 依次查询所有关联应用的日志
- ✅ 可视化应用链路图
- ✅ 按应用分组展示日志
- ✅ 支持日志时间定位

---

## 🎯 使用场景

### 场景 1: 协议支付交易追踪

**交易类型**: `310011` (协议支付)  
**关联应用**: `sft-aipg` → `sft-trxqry` → `sft-pay`

**操作流程**:
1. 选择交易类型 `310011`
2. 输入 REQ_SN: `20260408090000019466514799`
3. 输入日志时间：`2026040809`
4. 点击"开始追踪"
5. 系统自动：
   - 从 `sft-aipg` 找到 TraceID
   - 查询 `sft-aipg`、`sft-trxqry`、`sft-pay` 的所有相关日志
   - 按应用分组展示

### 场景 2: 批量协议支付交易追踪

**交易类型**: `310016` (批量协议支付)  
**关联应用**: `sft-aipg` → `sft-trxqry` → `sft-batchpay`

## 🖥️ 页面功能

### 查询条件

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 交易类型 | 下拉选择 | ✅ | 显示交易代码、名称和关联应用 |
| REQ_SN | 文本输入 | ✅ | 交易序列号 |
| 日志时间 | 文本输入 | ❌ | 格式：YYYYMMDDHH (10 位数字) |

### 结果展示

#### 1. 应用链路图
```
[1] sft-aipg    →    [2] sft-trxqry    →    [3] sft-pay
  15 条日志              12 条日志               8 条日志
```

- 显示所有关联应用
- 每个应用显示日志条数
- 点击应用可快速跳转

#### 2. 应用日志详情
- 按应用分组展示（Tab 切换）
- 每个 Tab 显示该应用的所有日志
- 支持展开查看完整日志内容
- 高亮显示包含 REQ_SN 的行

## 📊 查询逻辑

```
1. 用户选择交易类型 (如 310011)
       ↓
2. 系统加载关联应用列表
   (sft-aipg → sft-trxqry → sft-pay)
       ↓
3. 从第一个应用 (sft-aipg) 根据 REQ_SN + 时间找到 TraceID
       ↓
4. 使用该 TraceID 依次查询所有关联应用的日志
       ↓
5. 按应用分组展示结果
```

## ⚠️ 注意事项

1. **应用顺序**: 关联应用的顺序很重要，第一个应用作为入口应用
2. **日志时间**: 建议指定日志时间，可以加快查询速度
3. **TraceID 一致性**: 所有关联应用必须使用相同的 TraceID
4. **权限要求**: 确保 nginx worker 用户对所有日志目录有读取权限

- **响应式设计**: 适配各种屏幕尺寸

- **Tab 切换**: 按应用分组查看日志

- **空状态提示**: 未找到日志时友好提示

## 🔍 与日志追踪查询的区别

| 功能 | 日志追踪查询 | 交易类型追踪 |
|------|-------------|-------------|
| **查询方式** | 选择单个应用 | 自动加载多个应用 |
| **TraceID 提取** | 从选定应用提取 | 从入口应用提取 |
| **结果展示** | 所有日志合并 | 按应用分组 |
| **适用场景** | 精确定位某应用 | 查看完整链路 |
| **应用数量** | 1 个 | 多个（配置决定） |

## 📖 使用指南

### 快速开始

1. 访问 http://172.16.2.164:8083/transaction-trace
2. 选择交易类型（如 310011）
3. 输入 REQ_SN
4. 输入日志时间（可选，格式：2026040809）
5. 点击"开始追踪"
6. 查看应用链路图和日志详情

### 查看特定应用日志

1. 在应用链路图中点击应用名称
2. 或在 Tab 栏切换应用
3. 查看该应用的所有日志
4. 展开查看完整内容
